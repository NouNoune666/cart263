https://leahiscoding.github.io/CART263/finalProject/
Leah's project is honestly one of the most impressive ones I've seen. The website itself is clean, well-designed, and easy to navigate, which already sets it apart. The project offers three webcam filters powered by ML models: one tracks your face, one tracks your hands and fingers, and the last one reacts to sound. What surprised me is how little code the ml5.js face mesh model actually needs to run, just a few lines and it's detecting 468 facial landmarks. That discovery sent me down a rabbit hole. I didn't know ml5.js existed before this, and it turns out it can do a lot more than what Leah used here. It offers body pose estimation, image recognition, and even the possibilty of training your own neural networks. It's definitely bookmarked now. The Perlin noise was another thing I hadn't encountered before and it looks both accessible and fun to experiment with.On the technical side, what stood out most was the showCanvasIn() function, which moves a single canvas between page sections instead of reloading everything. Overall, it’s an impressive work with a cohesive aesthetic.

https://philippe-bn.github.io/cart263/projects/project-2/

The next project that caught my eye was Phil and Ben's 3D landscape. I almost didn't recognize it at first as I'd seen Phil's early process and it had evolved so much. Since I built my own 3D project, I was particularly curious about how all this worked under the hood. The landscape is huge and beautiful, and flying through it with WASD actually feels like being in a video game or like being one of the birds. The heightmap-based terrain also felt familiar from my experience with Unreal Engine. I had two main questions about the code. First, how were the flowers placed so precisely on the terrain? I believe that the answer is that each flower's random X and Z position gets checked using the heightmap array to make sure it lands in the right place.  Then getTerrainHeight() finds the exact Y position on the slope, and the stem is sunk just a bit into the ground so nothing floats. Second, how do all these birds avoid flying into mountains in real time? This seems to be done using the GPU which explains how it’s able to process so many assets without lagging or crashing. I don’t fully understand the logic behind it so I’ll have to ask Phil! 

https://emilebedard.github.io/cart263/Projects/project2/interactive_bike_garage_ii/index.html
Finally there's Emile's project, which I enjoyed partly because it's a continuation of his project one. He has such a clean aesthetic, and the timing is personally relevant because for project one I was starting to ride my bike in winter, and now for project two I'm taking out my summer bike, so the site is actually useful to me.
What interested me most in the code is: 1- the memory aspect: my bike "bikey" was still saved from last time, which works through the browser's localStorage, keeping my data persistent between visits. 2- The button animations: hovering over each button triggers a pre-built Blender animation (the cabinet door opens, the toolbox lifts), which plays forward on mouseover and reverses on mouseout by flipping the timeScale to -1. 3- The Blender import: the entire garage scene, including the camera, was exported as glb files and loaded with Three.js's GLTFLoader like in my project. 4- The movementL the camera follows your mouse using a parallax effect, normalizing the cursor position to a -1 to +1 range and smoothly lerping the camera toward the target offset each frame using a dampening factor.
